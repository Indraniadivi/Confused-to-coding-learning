Problem Domain & System Concept

1. Problem Domain
Many beginners who want to learn programming or mobile app development struggle with understanding technical concepts such as APIs, variables, databases, and application architecture. Traditional learning resources often rely heavily on documentation, tutorials, or lectures that introduce these concepts in an abstract way. As a result, new learners may find it difficult to connect theoretical knowledge with practical implementation.

Another challenge is that beginners often feel overwhelmed when attempting to build their first application because they do not have a clear mental model of how software components work together.

This project addresses this problem by introducing a gamified learning approach that teaches users how a real application works while guiding them through the process of building one.

Instead of presenting programming concepts purely through technical explanations, the system introduces them using a visual and metaphor-based framework that simplifies complex ideas into familiar real-world concepts.

The goal is to help learners understand how application components interact while gradually introducing the technical terminology required for real development.

2. System Overview

The project proposes a gamified learning guide that teaches users how a modern application works through structured missions.

In this system, the learner progresses through a series of guided tasks where they gradually build and understand the components of an application. Each mission introduces a new concept, such as data storage, API interaction, or user input.

The learning guide also includes prompt-based assistance, where users are given structured prompts they can use with any AI tool to ask questions, generate code examples, or clarify concepts. This approach encourages learners to actively explore solutions rather than passively follow instructions.

By the end of the learning journey, users should have a conceptual understanding of how an application is built and how different system components work together.

3. Storage Room Metaphor

To simplify complex technical ideas, the system introduces a Storage Room metaphor.

In this metaphor, an application is represented as a storage room where different elements store, move, and organize information.

This metaphor helps beginners visualize how data flows through a system.

For example, when a user logs food information into a calorie tracking application, the system receives the input, processes it, stores it, and then calculates nutritional values. Instead of explaining these processes in purely technical terms, the metaphor presents them as actions happening inside an organized storage system.

This approach allows learners to build an intuitive understanding before learning the underlying technical details.

4. Glossary of Metaphors

The following table shows how technical concepts are represented in the storage room metaphor:

| Technical Concept | Metaphor          |
| ----------------- | ----------------- |
| Variable          | Storage Bin       |
| Database          | Storage Warehouse |
| API               | Supplier          |
| Function          | Worker            |
| User Input        | Incoming Package  |
| Processing Logic  | Sorting Station   |

This glossary allows learners to translate between the simplified metaphor and the real technical terminology used in software development.

5. AI Prompt Learning Workflow

AI Prompt Learning Workflow

The learning guide introduces key concepts involved in building an application but does not attempt to explain every technical detail. Instead, the guide provides structured prompts that learners can use with external AI tools to explore topics in more depth.
These prompts act as guidance for learners who want additional explanations, examples, or code snippets while working through the learning modules.

For example, while learning about application architecture, the guide may introduce the three core components of an app:

Frontend – the user interface where users interact with the application

Backend – the logic that processes requests and manages data

Database – the system used to store application data

To learn more about these topics, learners may be given prompts such as:

“Explain how a backend server communicates with a mobile app.”

“Show a simple database structure for storing calorie tracking data.”

“What is the difference between frontend and backend?”

This approach encourages learners to explore concepts independently while using AI tools as a learning assistant.
